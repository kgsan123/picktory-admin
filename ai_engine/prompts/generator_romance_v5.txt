## 좋은 예측이란 (가장 중요)

연애 예능의 재미는 "누가 누구를 좋아하나"의 삼각관계·짝사랑·반전.
좋은 예측 = 방영 전 팬들이 "난 무조건 영자가 광수 선택해"라며 댓글로 싸우는 질문.

스스로 물어보세요: "이 질문에 팬들이 확신을 갖고 한쪽을 고르며 논쟁할까?"
아니라면 버리세요.

2가지 필수 조건:
1. 선택지는 [이번 회차 등장인물]의 실제 이름 (절대 "남자/여자/A" 금지)
2. 한 장면으로 즉시 판별 (선택 발표, 데이트 상대 호명 등)
※ 확률·퍼센트는 쓰지 않습니다. 선택지에 % 넣지 마세요.

가장 인기 있는 질문 유형 (팬들이 제일 많이 싸움):
- 삼각관계: "[A]는 [B]와 [C] 중 누구를 선택할까?"
- 짝사랑의 결말: "[A]의 마음을 [B]도 똑같이 받아줄까? → [B] 선택 / 다른 사람 선택 / 거절"
- 의외의 선택: "이번 회차 가장 의외의 커플로 묶일 두 사람은?"

## 예측 형식 (반드시 이 형식)

좋은 예시:
Q. 영자의 최종 선택은?
1. 광수
2. 영호
3. 상철
4. 아무도 선택 안 함

## 나는 SOLO 회차별 반복 이벤트

컨텍스트에서 어느 이벤트인지 파악하고 그 이벤트의 결과를 예측하세요.

**첫인상 선택 회차**:
- "첫인상 선택에서 아무도 선택받지 못할 [남/여]솔로는?" → 이름들 + 아무도 없음
- "[이름]의 첫인상 선택은?" → 이름들

**당일치기 여행 선택 회차**:
- "[이름]이 당일치기 여행에 선택할 파트너는?" → 이름들

**중간/최종 선택 회차**:
- "[이름]의 최종 선택은?" → 이름들 + 기권

**돌직구 대화 / 갈등 상황**:
- "[이름]이 [이름]에게 돌직구를 날릴 때, [이름]의 반응은?" → 솔직인정 / 화냄 / 회피 / 울음

## 하트시그널 / 기타 연애 프로그램

- 시그널 전송: "[이름]이 보낸 시그널의 수신자는?" → 이름들
- 전화 / 문자: "[이름]에게 전화가 올 사람은?" → 이름들

## 컨텍스트 활용

[이번 회차 등장인물] → 반드시 이 이름만 선택지에 사용
[뉴스 기사] → 이번 회차 이벤트 파악 → 그 이벤트의 결과를 예측 질문으로
[커뮤니티 반응] → 팬들이 주목하는 인물·상황 파악

컨텍스트에서 이벤트를 파악할 수 없으면 일반적인 연애 프로그램 이벤트로 대체:
- "이번 회차 첫 커플 성사의 주인공은?" → 이름들 + 없음
- "이번 회차 가장 많이 언급된 솔로는?" → 이름들

## 클로징(결과 확인 시점) — 각 예측에 "closing" 지정
- "next" = 다음 회차에 결과 확인 (대부분의 예측: 첫인상 선택, 데이트 상대, 회차별 선택 등)
- "finale" = 시즌 마지막에만 확인 (최종 커플, 최종 선택 우승자 등)
판단 안 서면 "next".

## 절대 금지

- YES/NO 형식 및 이진 반의어 (남자/여자, 진입/미진입, 성공/실패)
- 선택지 반드시 3개 이상
- [이번 회차 등장인물]에 없는 이름
- "방영 후 확인" (구체적 장면 명시 필수)
- 내면 심리 예측 ("사랑을 느낄까?")
- 모든 출력은 한글·영문·숫자만. 한자 금지.

## 이름이 없을 때 대체 전략

[이번 회차 등장인물]이 비어 있으면:
- 이름 없이 가능한 형식으로만 예측 생성
- ✅ "이번 회차 첫 커플 미션 성사 커플 수는?" → 0쌍 / 1쌍 / 2쌍 이상
- ✅ "이번 회차 돌발 퇴소자가 나올까?" → 없음 / 1명 / 2명 이상
- ❌ "A가 B를 선택할까?" (이름 없이 불가)
