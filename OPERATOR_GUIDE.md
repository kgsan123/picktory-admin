# PICKTORY 운영 가이드

이 도구를 가장 잘 활용하는 방법. 운영자(어드민)용 실전 가이드.

---

## 1. 한눈에 보는 사용 흐름

```
회차 방영 → [프로그램] 탭에서 "방영된 회차" 입력 → 예측 생성
        → [예측] 탭에서 검토 → 좋은 것 게시
        → 다음 회차 방영 → 정답 자동 확정(또는 수동 지정)
```

핵심 원리: **방금 방영된 N회를 입력하면, AI가 N+1회(다음 회차)용 예측을 만든다.**
정답은 N+1회가 방영된 뒤에 확정된다.

---

## 2. 프로그램 등록·설정 (여기서 품질의 80%가 결정됨)

[프로그램] 탭 → 각 카드의 **[컨텍스트·설정]** 펼치기.

| 항목 | 왜 중요한가 | 입력 팁 |
|------|-----------|---------|
| **카테고리** | 카테고리별로 다른 예측 프롬프트가 작동 | 연애/서바이벌/드라마/예능/음악 정확히 |
| **방송사·방영 요일·시각** | 자동 스케줄링 + 마감 시각 계산 | 정확히 넣을수록 자동 운영이 잘 됨 |
| **프로그램 형식 설명** | AI가 쇼의 규칙을 이해 → 엉뚱한 예측 방지 | 아래 예시 참고 |
| **출연자 명단** ⭐ | 뉴스에 이름이 안 나오는 쇼의 실명 선택지 확보 | 쉼표/줄바꿈 구분으로 한 번만 입력 |

**프로그램 형식 설명 예시:**
- 나는 SOLO: `같은 기수의 남녀 솔로가 만나는 형식. 같은 기수끼리만 선택. 하트시그널 등 다른 프로그램 이름 사용 금지.`
- 냉장고를 부탁해: `요리사 두 명이 대결하는 형식. 이번 회차 대결자는 컨텍스트에서 파악.`

**출연자 명단 — 일일이 찾지 마세요 (자동 추천):**
설정의 **[🔎 출연자 자동 추천]** 버튼을 누르면 나무위키·뉴스에서 출연자를 모아 칸을 채워준다.
→ **한 번 훑어보고 [설정 저장]** 하면 끝. (자동 추천은 후보일 뿐, **저장한 명단만** 예측에 쓰여 100% 운영자 확인 보장)
```
예) 박찬양, 김명주, 안시현, 나현진, 위준승, 변지영   ← 자동 추천 결과 예시
```
- 자동 추천이 빗나가거나 비면 직접 입력/수정하면 된다.
- 나는 SOLO처럼 기수가 많은 쇼는 현재 기수만 남기고 정리.
- 한 번 저장하면 시즌 내내 재사용된다. **이게 예측 품질에 가장 큰 차이를 만든다.**

---

## 3. 프로그램 유형별 핵심

| 유형 | 출연자 명단 | 비고 |
|------|:---:|------|
| **연애 (나는SOLO, 하트시그널, 넷플릭스 연애)** | **필수** | 명단 없으면 "누가 누구를" 예측 불가 |
| **서바이벌 (피지컬100, 오디션)** | **필수** | 참가자 명단 입력 |
| **드라마** | 권장 | 주요 배역명 입력하면 좋음 |
| **예능 (냉부, 런닝맨)** | 권장 | 고정 출연진 입력 |
| **음악 (인기가요, 뮤뱅)** | 불필요 | 차트가 자동 수집됨 |

---

## 4. 예측 생성 — 잘 만드는 법

1. **타이밍**: 방영 직후 말고 **2~4시간 뒤**에 생성하라. 후기·시청률 기사가 쌓여야 컨텍스트가 풍부해진다.
2. **"방영된 회차"**: 방금 끝난 회차 번호를 넣는다 → 다음 회차 예측 생성.
3. **컨텍스트 보강**(선택): 펼치기의 "이번 회차 핵심 내용"에 줄거리·사건을 넣으면 더 정확해진다.
4. 생성 결과 메시지를 확인:
   - ✅ `N개 예측 생성` → [예측] 탭에서 검토
   - ⚠️ `수집 정보 부족` → 출연자 명단 또는 핵심 내용을 직접 입력 후 재시도
   - ❌ `AI 일일 토큰 한도 초과` → 한도 회복(수십 분~) 대기 또는 Groq 업그레이드
   - ❌ `품질 기준 미통과` → 출연자 명단/컨텍스트 보강 후 재시도

---

## 5. 예측 검토·게시·관리

[예측] 탭. 상단 상태 필터(개수 표시): **검토 대기 · 게시됨 · 거부됨 · 전체**

- **검토 대기(draft)**: 새로 생성된 예측. 좋은 것 **게시**, 별로면 **거부**.
- **게시됨(published)**: 사용자에게 나가는 예측. 여기서 관리:
  - **정답 저장** — 결과가 나온 뒤 실제 일어난 선택지 지정
  - **게시 취소** — 잘못 게시한 것 되돌리기
  - **판정 취소(정정)** — 이미 넣은 정답 수정
- **거부됨(expired)**: 복원 가능

게시 전 체크: 선택지가 실제 이름인가? 질문이 다음 회차에 명확히 판별되는가?

---

## 6. 정답 확정 (검증)

- 각 예측에는 **마감 시점**이 있다:
  - 🔚 **다음 회차** — 대부분. 다음 회차 방영 후 결과 확인.
  - 🔚 **최종화** — 우승자·최종 순위 등. 시즌 끝에 확인.
- **자동**: 매일 11시(KST) 시스템이 방영 끝난 회차의 미판정 예측을 자동 재검증한다(오케스트레이터 구동 시).
- **수동/즉시**: [예측] 탭의 **"🔁 지금 pending 재검증"** 버튼으로 바로 돌릴 수 있다.
- **최종화 예측**은 자동 만료되지 않으니, 시즌이 끝나면 **정답 저장**으로 직접 확정한다.

---

## 7. 자동 운영 vs 수동 운영

**자동 (권장, 서버에서 `python orchestrator.py` 상시 구동):**
- 방영 30분 후 자동으로 감지→수집→검증→생성
- 매일 11시 pending 재검증 sweep
- 매주 일요일 신규 프로그램 자동 발견
- 실패 시 Discord 알림

**수동 (어드민에서 직접):**
- 회차 방영 후 [프로그램] 탭에서 생성 → [예측] 탭에서 검토·게시
- 다음 회차 후 "지금 재검증" 또는 정답 직접 지정

---

## 8. 품질을 높이는 5가지 핵심

1. **출연자 명단을 꼭 넣어라** — 실명 선택지 = 재미있고 판별 가능한 예측의 전제.
2. **프로그램 형식 설명을 써라** — 쇼 규칙을 알려줘야 엉뚱한/다른 프로그램 예측을 막는다.
3. **방영 2~4시간 뒤 생성** — 후기가 쌓인 뒤가 컨텍스트가 풍부.
4. **카테고리를 정확히** — 잘못된 카테고리 = 잘못된 프롬프트.
5. **게시 전 한 번 더 본다** — 선택지가 추상적이거나(남자/여자) 판별 모호하면 거부.

---

## 9. 문제 해결 (생성 실패 메시지별)

| 메시지 | 원인 | 대응 |
|--------|------|------|
| AI 일일 토큰 한도 초과 | Groq 무료 100k/일 소진 | 수십 분~ 대기 / Dev Tier 업그레이드 |
| 수집 정보 부족 | 뉴스·후기 부족 | 출연자 명단 + 핵심 내용 직접 입력 |
| 품질 기준 미통과 | 생성물이 필터 탈락 | 출연자 명단/컨텍스트 보강 후 재시도 |
| AI 응답 형식 오류 | 모델 일시 오류 | 그냥 다시 시도 |

---

## 10. 한계·주의

- **Groq 무료 한도 100k 토큰/일** ≈ 하루 생성 ~16회. 추적 프로그램이 많으면 금방 소진된다. 실사용은 Dev Tier 권장.
- **신규 발견**은 후보를 많이 띄운다(일반 예능·영화 일부 포함). 진짜 타겟(연애·서바이벌·드라마)만 골라 추적하라.
- **최종화 예측**은 시즌 종료 시 수동 판정 필요.
- 확률(%)은 쓰지 않는다. 게임의 정답은 "실제 일어난 선택지"다.

---

## 11. 추천 일상 루틴 (수동 운영 기준)

```
[방영일 저녁/다음날 오전]
1. 프로그램 탭 → 방영된 회차 입력 → 예측 생성
2. 예측 탭 → 검토 대기에서 좋은 것 게시, 나머지 거부
3. (다음 회차가 방영됐으면) "지금 재검증"으로 지난 예측 정답 확정
   - 자동 확정 안 된 것은 정답 직접 지정
4. 최종화 예측은 시즌 끝에 한 번에 정답 입력
```
